# 团队发言激励工具 - 规格说明

## 问题陈述

团队会议中同事们不愿主动举手发言，导致会议气氛沉闷、参与度低。需要一个有趣的工具，通过随机抽人发言 + 积分激励的方式，调动团队发言积极性，让开会变得更活跃有趣。

## 技术方案

- **前端**: React + TypeScript + Vite
- **后端**: Node.js + Express + TypeScript
- **数据库**: SQLite（轻量，无需额外部署）
- **动画**: Framer Motion
- **团队规模**: 10-20 人
- **数据**: 全局积分池，不区分会议场次

## 功能需求

### 1. 同事名单管理

- 添加/编辑/删除同事（姓名 + 头像颜色标识）
- 批量导入同事名单
- 每位同事显示：姓名、累计积分、发言次数

### 2. 随机抽人（核心功能）

- **抽取动画**: 名字快速滚动后减速停下，配合粒子特效和聚光灯效果
- **雨露均沾机制**: 已发言同事权重降低
  - 每次发言后，该同事被抽中权重降为原来的 50%
  - 当所有人都发言过后，权重自动重置
  - 当天已发言次数越多，权重越低
- **抽取结果展示**: 大字显示被抽中同事姓名，配合烟花动画
- 支持"再抽一次"（如该同事请假等特殊情况）

### 3. 随机鼓励积分

- 发言完成后，系统随机给出一个有趣的鼓励分数
- **有趣数字池**: 6、8、66、88、99、100、520、666、888、999 等吉利/有趣数字
- 积分揭晓时有开宝箱动画效果
- 显示一句随机鼓励语（如"太棒了！"、"说得好！"、"精彩发言！"、"你是全场最靓的仔！"）
- 积分自动累加到该同事的总积分

### 4. 积分排行榜

- **奖牌展示**: 前三名显示金🥇银🥈铜🥉奖牌，有闪光动画
- **排行榜列表**: 所有同事的排名、姓名、总积分、发言次数
- 排名变化时有上升/下降动画箭头
- 支持按时间段筛选（本周/本月/全部）
- 支持全屏投屏模式

### 5. 会议使用流程

1. 打开工具，进入会议主界面
2. 点击"开始抽人"按钮
3. 滚动动画播放，停下后显示被抽中的同事
4. 该同事发言
5. 发言结束后，点击"发放积分"
6. 积分揭晓动画，显示获得的分数和鼓励语
7. 积分自动累加，排行榜实时更新
8. 继续下一轮抽人，或结束会议

## 界面设计

### 整体风格
- 活泼有趣，游戏化感觉
- 明亮鲜艳的配色（橙色、黄色、蓝色为主色调）
- 圆角卡片、大字体、丰富动画
- 适合投屏到大屏幕展示

### 页面结构
1. **主页面 - 抽人界面**: 中央滚动区域 + "开始抽人"按钮
2. **积分发放页面**: 宝箱动画 + 分数和鼓励语展示
3. **排行榜页面**: 奖牌 + 排名列表，支持全屏
4. **管理页面**: 同事名单增删改查 + 数据导入导出

### 动画效果
- 抽人：名字快速滚动 → 减速 → 停下 + 聚光灯效果
- 积分揭晓：宝箱打开 → 数字跳动 → 最终分数放大显示
- 排行榜：排名变化位移动画，前三名闪光效果
- 按钮：hover 弹跳效果，点击涟漪效果

## API 设计

### 同事管理
- `GET /api/members` - 获取所有同事列表
- `POST /api/members` - 添加同事
- `PUT /api/members/:id` - 编辑同事
- `DELETE /api/members/:id` - 删除同事
- `POST /api/members/batch` - 批量导入

### 抽人与积分
- `POST /api/draw` - 执行随机抽人（考虑权重，返回被抽中同事）
- `POST /api/score` - 为指定同事发放随机积分（返回分数和鼓励语）

### 排行榜与记录
- `GET /api/leaderboard?period=week|month|all` - 获取排行榜
- `GET /api/history` - 获取发言历史记录

## 数据模型

### Member（同事）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 姓名 |
| avatar_color | string | 头像颜色标识 |
| total_score | number | 累计积分 |
| speak_count | number | 发言次数 |
| created_at | datetime | 创建时间 |

### SpeakRecord（发言记录）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| member_id | string | 同事 ID |
| score | number | 本次获得积分 |
| encouragement | string | 鼓励语 |
| created_at | datetime | 发言时间 |

## 实现步骤

1. **项目初始化**: 创建 monorepo 结构（client + server），配置 Vite + React + Express + TypeScript
2. **后端搭建**: Express 服务器 + SQLite 数据库 + 数据模型定义
3. **API 实现**: 同事管理、抽人、积分、排行榜的 RESTful API
4. **前端页面框架**: React Router 页面路由 + 整体布局 + 活泼主题样式
5. **同事管理页面**: 名单的增删改查 + 批量导入功能
6. **随机抽人核心功能**: 权重算法 + 滚动动画 + 聚光灯/烟花效果
7. **积分发放功能**: 有趣数字随机 + 宝箱揭晓动画 + 鼓励语
8. **排行榜页面**: 奖牌展示 + 排名动画 + 时间筛选 + 全屏模式
9. **动画打磨**: 所有交互动画细节优化
10. **联调测试**: 前后端联调，完整流程验证
